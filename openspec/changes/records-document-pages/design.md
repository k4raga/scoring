## Context

Сейчас документы доступны из карточки проекта и из страницы документов конкретного проекта. Это удобно для работы с одной записью, но не даёт общей навигации по документам как по отдельному слою.

Новая задача описывает слой подстраниц под `/records`, где документы рассматриваются как самостоятельная структура: месяцы, проекты, архивы, MD-документы и переходы внутрь.

## Goals

- Создать документный navigation layer под `/records`.
- Дать корневую страницу со списком документов по месяцам и проектам.
- Дать вложенную страницу со списком документов выбранного проекта/папки.
- Добавить обратные ссылки от документов к связанным задачам/карточкам проектов.
- Сохранить текущие prod-safe route rules: ссылки должны идти через приложение, а не через локальные пути.

## Non-Goals

- Не переносить все scoring-поля в документный слой.
- Не заменять текущую страницу проекта.
- Не делать редактор документов.
- Не принимать решения о типе закупки, рисках или заполнении scoring-полей.

## Proposed Shape

Предварительная форма, требующая подтверждения:

- `/records` — корневой индекс документного слоя.
- `/records/:recordId/documents` — текущая страница документов проекта может остаться вложенной страницей.
- `/records/:recordId/documents/:documentId` — текущий MD viewer остаётся страницей конкретного документа.
- На документных страницах должны быть ссылки назад к карточке проекта: `/records/:recordId`.

## Data Sources

Слой должен строиться из уже существующих данных:

- `record.id`, `record.projectTitle`, `record.publishedAt`;
- `record.documents`;
- `record.documentArtifacts`;
- `record.documentBlocks`;
- `record.workflow.extraction`;
- routes для архива, папки, оригиналов и MD.

## Risks / Trade-offs

- [Risk] `/records` может конфликтовать с текущим маршрутом `/records/:recordId` только концептуально, но технически корневой `/records` можно добавить отдельным route.
- [Risk] Индекс может стать шумным, если показывать все технические артефакты.
  - Mitigation: на корневой странице показывать архивы и MD, технические артефакты оставить внутри проектной деталки.
- [Risk] Термин "задача" может означать разные сущности.
  - Mitigation: зафиксировать точный тип обратной ссылки до реализации.

## Acceptance Notes

Реализация этой задачи должна начинаться только после уточнения Open Questions из `proposal.md`.
