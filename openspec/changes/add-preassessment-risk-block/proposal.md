## Статус

Черновик по скриншоту, короткому описанию пользователя и первичным уточнениям.

Основное предположение: нужен отдельный web-блок предоценки рисков для scoring/coding-записи, визуально и логически повторяющий табличный блок из текущего файла. Генерировать Excel-файл в этом change не нужно. Блок не является частью уже существующего блока критериев выбора победителя.

## Why

В текущем scoring-контуре есть этап предоценки, но нет явного структурированного блока, где сметчик и менеджер фиксируют риски участия, комментарии по обходу рисков, критичность и итоговое решение.

Без такого блока решение по предоценке остается в неструктурированном файле или в ручных комментариях, из-за чего его трудно использовать в карточке, AI-pass и последующей приемке участия.

## What Changes

- Добавляется новый блок `Предоценка (заполняет - сметчик)`.
- Блок отображает табличные строки риска с колонками:
  - `Параметр`;
  - `Комментарий менеджера и/или предложение о том можем ли обойти данный риск`;
  - `Критично/Не критично`.
- Для каждой строки риска фиксируется параметр, комментарий и статус критичности.
- Статус критичности выбирается из ограниченного набора значений, минимум: `Критично`, `Не критично`.
- Блок поддерживает фиксированную внешнюю ссылку на таблицу `База рисков`; отдельный пользовательский input для этой ссылки в первом UI не нужен. Позднее эта таблица может быть заменена или дополнена сервисной страницей.
- Добавляются итоговые строки:
  - `Решение по итогам предоценки`;
  - `Решение Александра`;
  - `Файл оценки`.
- `Решение Александра` остается персональным label в интерфейсе.
- `Решение по итогам предоценки` выбирается только из значений `Оценка` и `Не участвуем`.
- `Решение Александра` остается персональным label в интерфейсе и использует тот же выпадающий список `Оценка` / `Не участвуем`.
- Итоговые строки должны храниться отдельно от обычных строк риска и не смешиваться с перечнем параметров.
- В первом проходе перечень строк риска может заполняться вручную или из заготовленного шаблона; автоматическое извлечение из документов можно подключать позже через отдельный AI-pass.
- Визуальное представление должно быть web-аналогом исходного табличного блока в стиле текущего UI проекта, без желтого Excel-оформления и без генерации Excel-файла.

## Capabilities

### New Capabilities

- `preassessment-risk-block`: структурированный блок предоценки рисков scoring-записи, включая строки риска, комментарии, критичность, итоговые решения и связь с файлом оценки/базой рисков.

### Modified Capabilities

Нет.

## Impact

- Product/UX:
  - новый рабочий блок для этапа `Предоценка`;
  - табличное отображение рисков и итоговых решений;
  - более явная приемка решения по участию до перехода к дальнейшей оценке.
- Frontend:
  - новый UI-блок на странице scoring-записи;
  - контролы выбора критичности и итогового решения;
  - ссылка на базу рисков и/или файл оценки.
- Backend:
  - новая модель хранения строк предоценки и итоговых полей;
  - read/update API для сохранения блока в записи;
  - нормализация значений критичности и итогового решения.
- Coding/runtime:
  - структурированные данные блока доступны внутри scoring-записи;
  - будущая точка интеграции для AI-pass или шаблонного заполнения рисков.
- Workflow:
  - implementation должен начинаться после подтверждения точного места блока на странице, допустимых значений итогового решения и источника списка рисков.
