# FS-001 Extension

## Title

Agent-driven archive analysis for create-project flow

## Context

Это расширение к `FS-001` из [C:\Users\illki\Desktop\projects\scoring\.codex-workflow\future-scope.md](C:\Users\illki\Desktop\projects\scoring\.codex-workflow\future-scope.md).

Базовая история уже фиксирует сценарий:
- главная страница;
- `Добавить проект`;
- название проекта;
- выбор архива;
- создание записи;
- стартовая стадия `Скоринг`;
- выключенные поля там, где данных нет.

Это расширение добавляет главный продуктовый слой:
- агент должен не просто принять архив,
- агент должен развернуть архив,
- агент должен проанализировать документы внутри архива,
- агент должен извлечь значения и вставить их в поля карточки.

## Extended User Story

Пользователь создает проект с главной страницы через архив.

После загрузки:
1. система создает запись;
2. агент распаковывает архив;
3. агент проходит по всем релевантным документам;
4. агент извлекает значения для полей;
5. система вставляет найденные значения в карточку проекта;
6. поля без надежно найденных значений остаются выключенными.

## Extended Scope

1. Добавить агентный pass после загрузки архива:
   - распаковка архива;
   - инвентаризация файлов;
   - чтение документов;
   - извлечение фактов;
   - маппинг фактов в поля detail.
2. Анализ должен охватывать весь архив, а не только имя архива, внешние метаданные или один случайный файл.
3. Автозаполнение полей должно идти из содержимого документов, если нужные данные лежат внутри текста, таблиц или приложений.
4. Нужно явно различать состояния:
   - архив загружен;
   - архив распакован;
   - документы проанализированы;
   - поле заполнено агентом;
   - значение не найдено;
   - значение найдено, но недостаточно надежно для автоподстановки.
5. Поля без надежно извлеченного значения не должны автоматически становиться обычными пустыми editable-controls.

## Extended Acceptance

- После создания проекта запускается не только upload, но и агентный анализ архива.
- Агент действительно распаковывает архив и анализирует документы внутри него.
- Если значение присутствует в документах, система может подставить его в соответствующее поле.
- Если значение не найдено или найдено ненадежно, поле остается выключенным.
- Пользователь в detail видит результат анализа документов, а не просто факт наличия загруженного архива.
- Поведение остается совместимым с базовой историей `FS-001`: стартовая стадия новой записи — `Скоринг`.

## Functional Test Pack

`FT-FS-001-EXT-001` Agent pass starts after archive upload
- Создать проект через popup с архивом.
- Expect: после создания записи запускается отдельный агентный этап анализа архива.

`FT-FS-001-EXT-002` Archive is unpacked
- Использовать архив с несколькими файлами и вложенными документами.
- Expect: система распаковывает архив перед анализом, а не работает только с бинарным файлом как единым blob.

`FT-FS-001-EXT-003` Multiple documents are analyzed
- Использовать архив, где данные распределены по нескольким документам.
- Expect: агент проходит по нескольким документам, а не останавливается на первом найденном файле.

`FT-FS-001-EXT-004` Values are extracted from document content
- Использовать архив, где нужные значения находятся внутри текста документов.
- Expect: соответствующие поля detail автоматически заполняются по содержимому документов.

`FT-FS-001-EXT-005` Missing values remain disabled after analysis
- Использовать архив с неполным набором данных.
- Expect: поля без найденных значений остаются выключенными и визуально отличаются от заполненных.

`FT-FS-001-EXT-006` User sees analyzed result, not empty shell
- После завершения create-project flow открыть detail новой записи.
- Expect: карточка отражает результат агентного анализа документов: часть полей заполнена, часть выключена, а не вся форма просто пуста.

- Если потом захотите, следующим шагом это можно еще разделить на две отдельные истории:
