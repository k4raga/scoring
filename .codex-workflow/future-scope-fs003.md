# FS-003

## Title

Global fuzzy search from home

## User Story

Пользователь заходит на главную страницу и хочет найти проект через поиск.

Он может помнить:
- название проекта;
- часть описания;
- подрядчика.

Совпадение должно быть нестрогим: пользователь не обязан вводить точную строку целиком.

## Product Intent

Поиск на главной должен быть рабочим инструментом навигации по архиву, а не только формальным фильтром по точному совпадению.

Если пользователь вводит часть названия, часть описания или часть имени подрядчика, система должна находить релевантные проекты даже при неполном или неточном вводе.

## Scope

1. Зафиксировать главную страницу как точку входа в глобальный поиск по проектам.
2. Поддержать поиск как минимум по трем полям:
   - название;
   - описание;
   - подрядчик.
3. Сделать совпадение нестрогим:
   - partial match;
   - case-insensitive;
   - без требования полного точного текста.
4. Убедиться, что поиск работает как по текущей живой выдаче, так и по архивным проектам, попавшим в область поиска.
5. Сохранить различие между:
   - latest-preview/главная;
   - поисковой выдачей как отдельным состоянием страницы.

## Acceptance

- На главной доступен поиск.
- Пользователь может найти проект по части названия.
- Пользователь может найти проект по части описания.
- Пользователь может найти проект по части имени подрядчика.
- Совпадение не требует точной строки и полного регистра.
- Если найдено несколько релевантных проектов, они все попадают в выдачу.
- Если совпадений нет, показывается корректный empty-state поиска.
- Поиск не ломает остальные сценарии главной страницы.

## Functional Test Pack

`FT-FS-003-001` Search by partial title
- Открыть `/`.
- Ввести в поиск часть названия существующего проекта.
- Expect: проект появляется в поисковой выдаче.

`FT-FS-003-002` Search by partial description
- Открыть `/`.
- Ввести фрагмент текста из описания проекта.
- Expect: проект появляется в выдаче.

`FT-FS-003-003` Search by contractor
- Открыть `/`.
- Ввести имя подрядчика или его часть.
- Expect: все релевантные проекты этого подрядчика попадают в выдачу.

`FT-FS-003-004` Search is case-insensitive
- Повторить поиск в другом регистре.
- Expect: результат совпадает по смыслу и не зависит от регистра.

`FT-FS-003-005` Multiple matches are shown
- Использовать запрос, который совпадает с несколькими проектами.
- Expect: выдача показывает несколько релевантных результатов, а не только один.

`FT-FS-003-006` Empty state when nothing found
- Ввести запрос, которому ничего не соответствует.
- Expect: система показывает понятный empty-state поиска.

`FT-FS-003-007` Search does not break home browsing
- Очистить поиск после выдачи.
- Expect: главная возвращается в обычное состояние без поломки latest/filters/archive.

## Open Product Question

Нужно будет отдельно решить, насколько “нестрогое” совпадение нам нужно:
- только partial substring;
- или более широкий fuzzy match с опечатками, перестановками и нормализацией словоформ.

Сейчас история фиксирует минимальный рабочий вариант: нестрогий поиск без требования точного полного совпадения.
