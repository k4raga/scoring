# FS-002

## Title

Submitted projects filter from home

## User Story

Пользователь заходит на главную страницу и хочет посмотреть, какие проекты были поданы.

Для этого он:
- открывает главную;
- выбирает фильтр `Подано` в блоке последних проектов;
- ожидает увидеть все проекты со статусом `Подано` за текущий год.

## Product Intent

Этот кейс не про декоративную фильтрацию маленького блока `Последние проекты`, а про рабочую выборку.

Если пользователь на главной выбирает `Подано`, система должна показать полный список поданных проектов за текущий год, а не только несколько последних карточек из короткого preview-grid.

## Scope

1. Уточнить роль блока `Последние проекты` на главной:
   - preview-секция;
   - или точка входа в полноценную годовую выборку по стадии.
2. При выборе фильтра `Подано` дать пользователю доступ к полному набору проектов текущего года со стадией `Подано`.
3. Явно отделить:
   - ограниченный latest-preview;
   - полный filtered result-set по текущему году.
4. Сохранить канонический визуальный язык главной и не превращать экран в другой продуктовый сценарий без необходимости.
5. Зафиксировать, что выборка строится именно по текущему году, а не по всему архиву.

## Acceptance

- На главной доступен фильтр `Подано`.
- После выбора `Подано` пользователь видит все проекты текущего года со стадией `Подано`.
- Результат не ограничивается коротким latest-grid, если поданных проектов в году больше, чем в preview.
- Пользователь явно понимает, что смотрит выборку за текущий год.
- Если поданных проектов за текущий год нет, система показывает корректный empty-state именно для фильтра `Подано`.
- Остальные фильтры по стадиям не ломаются из-за этого сценария.

## Functional Test Pack

`FT-FS-002-001` Open submitted filter from home
- Открыть `/`.
- В блоке последних проектов выбрать фильтр `Подано`.
- Expect: активируется выборка по стадии `Подано`.

`FT-FS-002-002` User sees all submitted projects for current year
- Подготовить данные, где в текущем году есть несколько проектов со стадией `Подано`.
- Выбрать `Подано` на главной.
- Expect: интерфейс показывает весь набор поданных проектов за текущий год.

`FT-FS-002-003` Result is not truncated to preview size
- Подготовить набор, где поданных проектов за текущий год больше, чем помещается в обычный preview latest-grid.
- Выбрать `Подано`.
- Expect: пользователь получает доступ ко всем проектам, а не только к первым нескольким карточкам.

`FT-FS-002-004` Filter is scoped to current year
- Подготовить данные, где проекты со стадией `Подано` есть и в текущем, и в предыдущем году.
- Выбрать `Подано` на главной.
- Expect: в результат не попадают проекты прошлых лет.

`FT-FS-002-005` Empty state for submitted filter
- Подготовить набор, где в текущем году нет проектов со стадией `Подано`.
- Выбрать `Подано`.
- Expect: система показывает понятный empty-state для этого фильтра.

`FT-FS-002-006` Other stage filters still work
- Последовательно переключить `Скоринг`, `Предоценка`, `Оценка`, `Подано`, `Получен ответ`.
- Expect: фильтрация по каждой стадии продолжает работать и не ломает главную страницу.

## Open Product Question

Нужно будет отдельно решить, в каком виде показывать полный список:
- прямо на главной как expanded result-set;
- через отдельный режим/блок результатов;
- или через явный переход в годовую подборку.

Сейчас это не решено, поэтому история пока фиксируется как продуктовый scope, а не как готовое implementation-решение.
