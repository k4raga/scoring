# FS-004

## Title

Delete project

## User Story

Пользователь создал проект и хочет удалить его целиком.

Он должен иметь возможность удалить именно проект, а не отдельную задачу, поле или документ внутри карточки.

## Product Intent

Удаление проекта должно быть явным, управляемым и безопасным действием.

Пользователь должен:
- понимать, что удаляет весь проект;
- иметь защиту от случайного удаления;
- видеть предсказуемый результат после удаления.

## Scope

1. Добавить пользовательский сценарий удаления проекта.
2. Явно отделить удаление проекта от:
   - удаления отдельных полей;
   - удаления документов;
   - сброса изменений.
3. Дать пользователю точку входа в удаление проекта из карточки проекта.
4. Добавить подтверждение удаления, чтобы нельзя было удалить проект случайно одним кликом.
5. Определить, что именно удаляется вместе с проектом:
   - запись в реестре;
   - связанные данные карточки;
   - ссылки на документы;
   - архивные/служебные артефакты, если это нужно по продуктовой логике.
6. Определить, куда попадает пользователь после удаления:
   - на главную;
   - в месяц;
   - в предыдущий контекст.

## Acceptance

- Пользователь может инициировать удаление проекта из интерфейса проекта.
- Перед удалением система показывает явное подтверждение.
- Подтверждение однозначно говорит, что удаляется весь проект.
- После подтверждения проект удаляется из пользовательского интерфейса и из реестра.
- После удаления пользователь попадает в понятный следующий экран, без битого route и ошибки.
- Удаленный проект больше не появляется:
  - в detail-route;
  - в месячной выборке;
  - в главной выдаче.
- Если удаление невозможно, пользователь получает понятную ошибку, а проект не пропадает “наполовину”.

## Functional Test Pack

`FT-FS-004-001` Open delete action
- Открыть detail существующего проекта.
- Найти действие удаления проекта.
- Expect: действие удаления доступно как отдельный сценарий, не смешано со `Сбросить изменения`.

`FT-FS-004-002` Confirmation is required
- Инициировать удаление проекта.
- Expect: система показывает подтверждение перед фактическим удалением.

`FT-FS-004-003` Cancel deletion
- Открыть подтверждение удаления.
- Нажать отмену.
- Expect: проект не удаляется и остается доступным.

`FT-FS-004-004` Confirm deletion
- Подтвердить удаление проекта.
- Expect: проект удаляется и detail-route больше не остается рабочей страницей этого проекта.

`FT-FS-004-005` Project disappears from lists
- После удаления открыть главную и соответствующий месяц.
- Expect: проект отсутствует в списках и карточках.

`FT-FS-004-006` Redirect after delete is stable
- После подтверждения удаления проверить итоговую навигацию.
- Expect: пользователь попадает на понятную страницу без broken state.

`FT-FS-004-007` Failed delete is handled cleanly
- Смоделировать ошибку удаления.
- Expect: пользователь получает понятную ошибку, а проект остается доступным и консистентным.

## Open Product Questions

Нужно отдельно решить:
- удаление должно быть жестким или через soft-delete;
- нужно ли удалять связанные архивы и служебные артефакты;
- нужен ли recover/undo сценарий после удаления.

Пока эта история фиксирует минимальный продуктовый контракт: пользователь должен иметь возможность удалить проект целиком через явный подтвержденный сценарий.
