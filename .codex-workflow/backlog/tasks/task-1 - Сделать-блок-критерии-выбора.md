## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Нужно спроектировать и реализовать новый блок критериев выбора для менеджера по тендерам.

Сценарий: менеджер изучает документацию закупки и фиксирует критерии, по которым организатор тендера выбирает победителя. Критерии бывают ценовые и неценовые, также существуют дополнительные требования. Для каждого пункта важно понимать, закрываем ли мы его полностью, частично или не закрываем, и каким образом.

Блок должен располагаться внизу страницы под блоком "Информация по тендеру". Одна строка блока соответствует одному критерию или требованию. Текущая скрытая реализация критериев не считается корректной основой и должна быть удалена или заменена.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [x] #1 Блок критериев выбора отображается под блоком "Информация по тендеру".
- [x] #2 Одна строка блока соответствует одному критерию или одному дополнительному требованию.
- [x] #3 Поддерживаются группы: ценовой критерий, неценовой критерий, дополнительное требование.
- [x] #4 Для ценовых и неценовых критериев поддерживается вес, если он указан в документации.
- [x] #5 Для каждой строки обязателен статус закрытия: полностью закрываем, частично закрываем или не закрываем.
- [x] #6 Пользователь может визуально добавлять и удалять строки критериев.
- [x] #7 Порядок строк соответствует порядку добавления; отдельная ручная сортировка на первом этапе не требуется.
- [x] #8 Вес критерия хранится и отображается как процент.
- [x] #9 Критерии сохраняются как структурированные данные, пригодные для scoring input и будущего `AI-pass`.
- [x] #10 Текущая скрытая реализация критериев удалена или заменена и не управляет новым блоком.
- [x] #11 Тестовый extractor пишет критерии в новую модель в первом implementation pass.
- [x] #12 Старые criteria-данные не переносятся в новую модель.
